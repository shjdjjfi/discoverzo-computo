# Reproducibility guide

This repository is the complete Computo submission source and research artifact for **Discovering active subspaces from noisy function values**. It contains the article source, installable implementation, deterministic benchmark instances, repetition-level results, generated summaries, figure/table programs, tests, and continuous-integration workflows.

## 1. Reproducibility levels

The artifact supports three increasingly expensive levels.

### Level A: render from committed results

This is the default article build. It does not rerun optimization experiments.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e ".[test]"
python scripts/make_manuscript_assets.py
quarto add computorg/computo-quarto-extension@v1.0.4 --no-prompt
quarto render
```

Quarto writes the official Computo HTML and PDF outputs to `_site/`. The manuscript reads committed CSV files and regenerates all displayed assets before rendering.

### Level B: tests and reduced end-to-end run

```bash
pytest
python scripts/validate_research_artifact.py
python scripts/validate_manuscript.py
python -m discoverzo.cli --config configs/quick.yaml --output results/quick
```

The quick configuration exercises benchmark generation, oracle accounting, subspace discovery, optimization, summaries, and plotting on a reduced grid.

### Level C: complete verified experiment

```bash
python scripts/run_verified.py
python scripts/paired_analysis.py
python scripts/make_manuscript_assets.py
```

The verified run is intentionally decoupled from document rendering. Its optimization portion is divided into deterministic restartable batches so that interrupted server jobs can resume without changing random instances.

## 2. Data provenance

No private data, large language model, neural surrogate, or proprietary benchmark is used. The experimental data consist of:

1. public analytical Sphere, Ellipsoid, Rosenbrock, and Rastrigin formulas from the BBOB literature;
2. seeded orthogonal embeddings, translations, anchor points, and approximation strengths generated by the repository;
3. additive Gaussian oracle noise generated from the fixed seed ledger.

Static inputs are stored in `datasets/generated_inputs/`. Their metadata and all RNG seeds are stored as CSV files. Repetition-level outputs are duplicated under `results/raw/` and `datasets/raw_outputs/`; summary outputs are duplicated under `results/summary/` and `datasets/summary_outputs/`.

The default benchmark is controlled rather than an official COCO performance submission: it uses public BBOB formulas but not every official COCO instance transformation or target-based post-processing step. `environment-coco.yml` and the optional adapter in `src/discoverzo/benchmarks.py` support a future full COCO campaign in a compatible environment.

## 3. Exact query accounting

Every function evaluation is recorded by the oracle ledger. At each anchor, a cross-fitted estimator with `m` directions on each side uses

```text
2 calls/direction x m directions x 2 independent sides = 4m calls.
```

The moment-calibration experiment compares equal-query estimators at 16 calls per anchor:

- cross-fitted: two independent four-direction batches;
- self outer product: one eight-direction batch;
- trace-adjusted self outer product: the same eight-direction batch followed by the stated heuristic correction.

For optimization, discovery calls are deducted from the common total budget before any restricted update is taken. The full-dimensional comparator receives the same total number of evaluations.

| Study | Main settings | Repetitions | Raw rows |

| Moment calibration | `d=12`, `r=2`, seven anchor counts | 20 | 420 |

`configs/verified.yaml` is the sole configuration used for the manuscript. `configs/high_power.yaml` is supplied for future server-scale confirmation but is not used in any reported table or figure.

## 5. Statistical summaries

- Estimation curves: arithmetic mean and standard error.
- Optimization distributions: median, interquartile range, all repetition points, and 5th/95th percentile whiskers.
- Paired comparisons: within-instance difference in `log10` normalized regret relative to full-dimensional ZO.
- Uncertainty: percentile interval from 20,000 paired bootstrap resamples with a fixed seed.

The optimization study has ten paired repetitions per setting. Intervals are reported as uncertainty summaries; the article does not convert them into claims of universal dominance.

This script produces:

## 7. Environment

The primary environment is defined by:

- `pyproject.toml`;
- `requirements-lock.txt`;
- `environment.yml`;
- `environment-coco.yml` for the optional compiled COCO dependency.

The validated environment and executed commands are recorded in `docs/VALIDATION.md`. The article itself evaluates only a lightweight package-version chunk; expensive experiments are never triggered automatically during rendering.

## 8. Integrity and validation

Run

```bash
python scripts/validate_research_artifact.py
python scripts/validate_manuscript.py
```

The first command checks row counts, missing values, schemas, and hashes for the four primary raw-result files. The second checks citations, unique cross-reference identifiers, and the presence of all seven figures in three formats. `ARTIFACT_MANIFEST.json` and `SHA256SUMS` provide file-level hashes for the final submission package.

## 9. Scientific boundaries

The repository distinguishes proven statements from computational stress tests:

- matrix concentration is proved for bounded sketches; clipping requires an additional bias term;
- the exact rank proposition covers a calibrated spectral threshold, whereas the implemented normalized eigengap is empirical;
- the optimization transfer theorem covers projected stochastic two-point gradient descent, not the Adam-style stabilization used in experiments;
- Rosenbrock and Rastrigin experiments lie outside the convex global theorem and are explicitly treated as stress tests;
- the lower bound establishes an ambient-dimensional discovery cost in one Gaussian rank-one family, not a universal direct-sum lower bound for every optimization class.

These boundaries are repeated in `docs/theory_audit.md` and in the manuscript discussion.
